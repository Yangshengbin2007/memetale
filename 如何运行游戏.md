# 如何运行 memetale RPG 游戏

## ⚠️ 重要：必须在项目根目录运行！

**错误示例：**
```bash
cd src/game/launcher
javac Main.java  # ❌ 这样会失败！
```

**正确方法：**

### 方法 1：使用 run.command（推荐）⭐

1. **在 Finder 中**：双击 `run.command` 文件
   - 首次运行：右键点击 → "打开"

2. **在终端中**：
```bash
cd /Users/415330/Documents/GitHub/Beather/Jerry-advprogramming2026-Individual-project-Repo
./run.command
```

### 方法 2：手动编译和运行

**步骤 1：进入项目根目录**
```bash
cd /Users/415330/Documents/GitHub/Beather/Jerry-advprogramming2026-Individual-project-Repo
```

**步骤 2：编译（从项目根目录）**
```bash
javac -encoding UTF-8 -d out -sourcepath src src/game/launcher/Main.java
```

**步骤 3：运行（从项目根目录）**
```bash
java -cp out game.launcher.Main
```

## 📋 为什么不能在 launcher 目录下编译？

1. **包结构**：`Main.java` 在包 `game.launcher` 中
2. **依赖关系**：需要 `game.scene`、`game.model` 等其他包
3. **编译路径**：必须从项目根目录编译，指定 `-sourcepath src`

## ✅ 正确的目录结构

```
项目根目录/
├── src/
│   └── game/
│       ├── launcher/
│       │   └── Main.java  ← 主文件
│       ├── scene/          ← 依赖的包
│       ├── model/          ← 依赖的包
│       └── io/              ← 依赖的包
├── out/                    ← 编译输出目录
├── image/                  ← 资源文件
├── sound/                  ← 资源文件
├── music/                  ← 资源文件
├── Stickers/              ← 资源文件
└── run.command             ← 运行脚本
```

## 🎮 展示时使用

### 推荐方法：在终端运行（最可靠）⭐

**步骤：**
1. 打开 **终端** (Terminal.app)
2. 运行以下命令：

```bash

```

3. 等待编译完成（会显示 "✅ Compilation successful!"）

2. 找到项目文件夹

4. 等待游戏窗口打开

## ⚠️ 如果遇到问题

**编译失败：**
- 检查 Java 是否安装：`java -version`
- 确保在项目根目录运行
- 查看终端中的具体错误信息

**运行失败：**
- 确保资源文件夹存在：`image/`, `sound/`, `music/`, `Stickers/`
- 检查 `out/` 目录中是否有编译后的 `.class` 文件
- 查看终端中的错误信息

**窗口不显示：**
- 检查是否有其他 Java 程序在运行
- 尝试关闭并重新运行
- 查看终端是否有错误信息
